# Awakening

![Awakening](../assets/awakening.png)

Writing software will never be the same. Although top engineers will always be solving novel problems, most of us - including myself - spend most of our time implementing features that might require some learning but aren't breaking new ground for the field.

I valued that my experiences would compound over time and make me a better engineer for the next project. However, I have to honestly acknowledge that generative models today can write better code faster in the majority of cases. At the very least the speed at which they can generate mostly correct code is not something I can match.

The bottleneck is no longer in writing code, but in my ability to architect solutions, review generated code, and effectively prompt these models. While I still plan to learn and practice coding as before, for outcome-oriented software the economics don't make sense to write code as before. I firmly belive that engineers that use generative models will replace the ones that don't and that people's ambition of what is possible with software will increase greatly as a result.

For this reason, I'm fully committing to augmenting my skills with generative models and building tools to enhance my productivity. I'll be sharing more about my approach and tools in future posts as I develop them.

For anyone beginning this journey, here are some essential resources I've found valuable:

* [Cursor](https://www.cursor.com/):
    * Currently the most sophisticated AI-powered code editor available

* [Windsurf](https://codeium.com/windsurf):
    * Another powerful AI-assisted development environment
    * While I experienced fewer breakthrough moments with Windsurf, it remains competitive with Cursor

* [Standard Library for AI Development](https://ghuntley.com/stdlib/):
    * Article that inspired this post
    * Excellent concept of building a library of chainable prompts for code generation

* [Aider Model Performance Rankings](https://aider.chat/docs/leaderboards/):
    * Comprehensive benchmarks of current AI coding models
    * Updated regularly with new model comparisons

* [Model Context Protocol Servers](https://github.com/modelcontextprotocol/servers):
    * A curated list of servers supporting the Model Context Protocol
    * Essential resource for server implementation and discovery
